" Comandos más comúnes de docker

docker pull {imagen} - descargar un hub de docker pero no correrlo

docker run [-d] {imagen} - correr una imánges de docker, si no está descargada la
descarga; -d 'detach' corre un contenedor sin mostrar los logs

docker images - muestra todas las imágenes descargadas

docker ps - puestra los procesoso, las imágenes que has sido ejecutadas en un
tiempo delimitado por el sistema

docker start {id|name} - iniciar una imágen especificando su id o nombre,
recupera el estado del contenedor

docker logs [-f] {id|name} - muestra todos los logs del conenedor; -f hará que
el proceso de impresión no termine y siga imprimendo los logs

docker exec (-it) {id|name} [command] - ejecuta un comando dentro de un
contenedor que ya está corriendo; -it hará una seción iterativa; un ejemplo de
comando puede ser 'sh' para ejecutar una shell

docker stop {id|name} - detiene un contenedor
